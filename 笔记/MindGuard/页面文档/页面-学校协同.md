# 页面：学校协同 DASHBOARD

## 页面定位
学校协同页面是为高校心理辅导员、运营老师（ops）设计的专业后台管理界面，作为风险事件的人工干预中枢，实现从"AI预警"到"真人协同"再到"行为闭环"的完整管理流程。该页面聚焦于处理由系统自动识别的需要人工介入的学生风险事件，为专业人员提供高效、安全的协同工作环境。

## 核心模块与入口

* **模块1: 风险预警队列 (Alerts Queue)**
  * **功能描述**: 以看板形式展示所有待处理、跟进中、已解决的匿名化风险事件，支持卡片式布局展示关键信息，包括风险等级（L1-L4）、触发原因关键词、事件时间、校区分布等核心要素，帮助辅导员快速识别需要优先处理的紧急情况
  * **交互说明**: 支持多维度筛选（按风险等级、校区、时间范围、状态过滤），灵活排序（按风险级别、时间戳、处理状态），智能搜索（关键词匹配），提供快速标记和批量处理功能

* **模块2: 事件详情视图 (Case Detail View)**
  * **功能描述**: 点击预警卡片后展示完整的脱敏事件信息，包括原始内容摘要、学生历史行为模式、AI风险判定依据、相关情绪打卡数据、过往干预记录等，为辅导员提供全面的决策支持
  * **交互说明**: 支持一键"开始跟进"、"标记误报"、"升级处理"等操作，提供时间线式的历史记录查看，支持添加专业评估笔记和处理建议

* **模块3: 真人任务指派 (Intervention Task Assignment)**
  * **功能描述**: 辅导员可通过此模块向学生创建和指派个性化的干预任务，支持从预设模板库选择或自定义任务内容，任务会同步到学生的任务看板中，形成完整的行为干预闭环
  * **交互说明**: 提供任务模板快速选择、截止时间设定、优先级标记、执行说明添加等功能，支持任务执行进度的实时跟踪和效果评估

* **模块4: 数据统计面板 (Statistics Panel)**
  * **功能描述**: 页面顶部展示关键运营数据，包括"待处理事件数"、"平均响应时长"、"本周已闭环数"、"干预成功率"等核心指标，帮助管理人员把握整体工作状态和效率

## AI / 协同逻辑

* **风险来源**: 风险信号主要来源于"树洞"模块的AI自动研判系统，通过自然语言处理技术识别帖子中的风险关键词、情绪异常模式、行为变化趋势等，形成结构化的风险评分和等级判定

* **协同流程**: 完整的协同闭环包含：AI自动识别风险 → 生成匿名预警并推送到辅导员工作台 → 辅导员评估风险等级 → 学生授权后查看详细信息 → 制定个性化干预方案 → 指派具体任务给学生 → 学生执行任务并提供反馈 → 辅导员评估干预效果 → 更新事件状态和记录经验

* **隐私保护**: 所有呈现给协同端的信息都经过严格的匿名化处理，只有在获得学生明确授权或在特定安全条件下才能追溯详细信息。系统采用"最小必要原则"，确保在有效干预和隐私保护之间取得平衡

## 涉及组件

* **FilterBar**: 数据筛选条组件，支持多条件组合筛选和快速搜索
* **StatsCard**: 统计卡片组件，用于展示关键运营指标
* **CaseCard**: 风险事件卡片组件，展示预警基本信息和状态
* **StatusBadge**: 状态标签组件，用不同颜色标识事件处理状态
* **AssignmentModal**: 任务指派弹窗组件，支持任务创建和指派操作
* **TimelineView**: 时间线视图组件，展示事件处理历史记录
* **RiskIndicator**: 风险等级指示器组件，直观显示风险级别

## 页面跳转关系

* **主页面结构**: 从风险预警队列列表点击条目，在右侧详情区域或新标签页打开完整的事件详情视图
* **任务指派流程**: 在事件详情视图中点击"指派任务"，弹出任务创建表单，支持从模板库选择或自定义任务内容
* **学生档案关联**: 在授权状态下，可跳转到学生的基础档案页面（脱敏信息）查看更全面的背景资料
* **历史记录查看**: 支持跳转到完整的历史干预记录页面，查看过往的干预方案和效果评估

## 数据需求与API

* **数据来源**:
  * `reports` - 存储风险预警和周报数据，包含风险摘要、等级判定和处理状态
  * `tasks` - 任务管理集合，记录真人指派任务的创建、执行和完成情况
  * `users` - 用户信息集合，提供学生基础信息和辅导员关联关系
  * `posts` - 树洞帖子集合，作为风险分析的主要数据源之一

* **关键API**:
  * `GET /api/collaboration/alerts`: 获取风险预警列表，支持多维度筛选、分页和排序功能
  * `GET /api/collaboration/alerts/{id}`: 获取单个预警事件的详细信息，包含完整的脱敏数据和历史记录
  * `PATCH /api/collaboration/alerts/{id}`: 更新事件状态（如：开始跟进、标记误报、已解决等处理状态）
  * `POST /api/tasks`: 创建并指派真人干预任务，支持任务模板选择和个性化设置
  * `GET /api/collaboration/statistics`: 获取协同工作的统计数据和运营指标
  * `POST /api/collaboration/authorization`: 处理学生授权请求，开通详细信息访问权限
